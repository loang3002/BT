Bài 3.1: Quan sát một số URL trên mạng

  1. Mở trình duyệt, nhập vào thanh địa chỉ một URL bất kỳ, ví dụ: https://en.wikipedia.org/
  2. Trình duyệt sẽ đổi tên miền en.wikipedia.org sang địa chỉ IP, kết nối tới máy server dựa trên IP và lấy về trang web. Bạn vào cửa sổ cmd, gõ lệnh: ping en.wikipedia.org, ghi lại địa chỉ IP máy server của wikipedia.
  3. Server sẽ gửi trang mặc định về cho trình duyệt, ví dụ https://en.wikipedia.org/wiki/Main_Page
  4. Di chuyển chuột vào các liên kết (ít nhất 5 liên kết) có trên trang web, quan sát trên thanh trạng thái (status bar) của trình duyệt (thanh ngang phía dưới của cửa sổ trình duyệt), ghi lại các URL mà bạn quan sát được.
  5. Thêm thông tin cổng và URL để quan sát kết quả, ví dụ nếu giao thức là https thì thêm cổng 443, giao thức http thì thêm cổng 80, ví dụ: https://en.wikipedia.org:443/wiki/Main_Page. Bạn sẽ thấy trang web vẫn hiển thị bình thường (không bị lỗi).
  6. Kiểm tra xem trang wikipedia có hỗ trợ giao thức HTTP hay không, bằng cách nhập vào URL sau: http://en.wikipedia.org:80/wiki/Main_Page
  7. Vào trang Tiki.vn, sử dụng trang web, quan sát và ghi lại giá trị của chuỗi truy vấn “query_string”. Ví dụ: https://tiki.vn/search?q=loa%20m%C3%A1y%20t%C3%ADnh

Bài Tập 3.2: Quan sát URL trên máy cục bộ

  1. Trên máy cục bộ, sử dụng phầm mềm Notepad tạo một tập tin HTML có nội dung bất kỳ, ví dụ: D:\index.html với nội dung là dòng chữ “Xin chao”
  2. Bấm đúp vào tập tin index.html để mở tập tin bằng trình duyệt, sẽ thấy dòng chữ “Xin chao” xuất hiện trên màn hình.
  3. Quan sát đường dẫn sẽ thấy URL có nội dung sau: file:///D:/index.html
  4. Tạo thư mục mới có tên là mywebsite, chuyển tập tin index.html vào trong thư mục vừa tạo, quan sát URL để hiểu thêm về đường dẫn thư mục. Ví dụ: file:///D:/mywebsite/index.html
